## Product Validation

### 4. Distributed migration readiness

Action:

- follow [`../distributed-operations.md`](../distributed-operations.md) on a distributed beta deployment
- confirm the Storage page shows migration bytes and the drain-complete signal

Expected:

- `Pending Local Bytes` and `Distributed Bytes` are visible
- `Local drain complete` only appears when local backlog is fully cleared
- demoting the last healthy active node is blocked while local drain is incomplete

Evidence:

- started the published bundle in distributed local-node mode with the shipped `release-images.env` and a bundle-local `.env.distributed`:
  - `STORAGE_BACKEND=distributed`
  - `STORAGE_DISTRIBUTED_ENDPOINTS=http://blobnode-a:9100,http://blobnode-b:9100,http://blobnode-c:9100,http://blobnode-d:9100`
  - `STORAGE_DISTRIBUTED_REPLICAS=3`
- setup was completed successfully with:
  - `desiredStorageBackend=distributed`
  - `distributedScope=local`
  - `runtimeStorageBackend=distributed`
  - `runtimeEndpointCount=4`
- the distributed node catalog came up healthy from the published bundle:
  - `blobnode-a`
  - `blobnode-b`
  - `blobnode-c`
  - `blobnode-d`
- local-drain and safeguard behavior were validated against the running published release:
  - with all four nodes in `maintenance`, a fresh object remained local and `migration-status` reported:
    - `pendingLocalObjects=1`
    - `pendingLocalBytes=39`
  - after promoting only one node to `active`, HarborShield blocked demoting that last healthy active node with `409 Conflict`:
    - `cannot move the last healthy active node out of service while 1 local objects (39 bytes) are still waiting for migration`
- migration then succeeded after reactivating the full node set:
  - `migratedCount=1`
  - post-migration `pendingLocalObjects=0`
  - post-migration `pendingLocalBytes=0`
  - post-migration `distributedObjects=3`
  - post-migration `distributedBytes=112`
- placement/readback validation succeeded on the migrated object:
  - bucket: `rc4-guard2-1774363706`
  - key: `guard2.txt`
  - placement matches recorded: `3`
  - downloaded payload: `hello from rc4 distributed guard status`
