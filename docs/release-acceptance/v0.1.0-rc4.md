# Release Acceptance: v0.1.0-rc4

Use this checklist to validate the published `v0.1.0-rc4` release exactly as an external operator would consume it.

This checklist is intentionally release-bundle based:

- use the published GitHub release assets
- do not run from a source checkout
- keep `release-images.env` exactly as shipped

## Preconditions

- start from a clean machine or clean Docker environment
- download the published `v0.1.0-rc4` release asset
- extract the bundle to a fresh working directory
- copy `.env.example` to `.env`
- rotate every placeholder secret in `.env`
- keep `release-images.env` unchanged for this pass

## Install Validation

### 1. Bundle integrity

Action:

- verify the downloaded artifact matches `HarborShield-v0.1.0-rc4-sha256.txt`

Expected:

- checksum matches
- extracted bundle contains:
  - `docker-compose.yml`
  - `.env.example`
  - `release-images.env`
  - `README.md`
  - `docs/`

Evidence:

- pending

### 2. Clean start

Action:

- start the stack from the extracted release bundle:

```bash
docker compose --env-file .env --env-file release-images.env up -d
```

Expected:

- images pull successfully from GHCR
- containers start without editing image references manually
- `docker compose ps` shows core services healthy
- `http://localhost/healthz` returns `{"status":"ok"}`

Evidence:

- pending

### 3. First-run operator pass

Action:

- execute the first-run flow using [`../first-run-checklist.md`](../first-run-checklist.md)

Expected:

- bootstrap login works
- forced password change works
- setup wizard completes
- first bucket, credential, upload, and audit review succeed

Evidence:

- pending

## Product Validation

### 4. Distributed migration readiness

Action:

- follow [`../distributed-operations.md`](../distributed-operations.md) on a distributed beta deployment
- confirm the Storage page shows migration bytes and the drain-complete signal

Expected:

- `Pending Local Bytes` and `Distributed Bytes` are visible
- `Local drain complete` only appears when local backlog is fully cleared
- demoting the last healthy active node is blocked while local drain is incomplete

Evidence:

- pending

### 5. S3 smoke on published build

Action:

- create an S3 credential from the UI
- validate bucket create, object upload, and object download against the running release

Expected:

- core documented S3 path works against the published bundle
- no install-specific regression appears versus source-based validation

Evidence:

- pending

## Recovery Validation

Expected:

### 7. Restore evidence

Expected:

Evidence:

- pending

## Exit Criteria

Mark this acceptance pass complete only if all are true:

- published bundle installs cleanly
- first-run flow succeeds from the published bundle
- distributed migration readiness signals behave as documented
- core S3 workflow succeeds on the published bundle
- backup and restore evidence is captured
