# HarborShield v0.1.0-rc1

Published: 2026-03-23

Release type:

- prerelease
- `single-node` is the candidate GA path
- `distributed` remains beta

## Highlights

- first tagged HarborShield prerelease published through GitHub Actions
- full release validation passes before publish
- versioned GHCR images are now produced for backend and frontend
- release bundles now ship with pinned image references and SHA-256 checksums
- malware scan mode is now configurable directly from the Malware section in the admin UI
- distributed beta now has hosted validation for live node admission and local-to-distributed migration

## Included In This Prerelease

- Docker Compose deployment bundle with `release-images.env`
- admin UI and API for object, bucket, audit, event, quota, policy, health, settings, and malware workflows
- S3 compatibility coverage documented in [`docs/s3-compatibility.md`](../s3-compatibility.md)
- automated fast CI, deep release validation, and tagged release publishing

## Operator Notes

- start the published bundle with:

```bash
docker compose --env-file .env --env-file release-images.env up -d
```

- rotate every secret in `.env` before exposing the stack beyond local testing
- `distributed` deployment remains beta and is not part of the GA promise for this release line
- malware scan mode can now be switched between `advisory` and `enforcement` in the Malware section
- the Storage page now reports `Pending Local Bytes`, `Distributed Bytes`, migration history, and a `Local drain complete` signal for distributed migrations
- the supported distributed day-2 runbook is documented in [`docs/distributed-operations.md`](../distributed-operations.md)
