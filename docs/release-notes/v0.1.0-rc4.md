# HarborShield v0.1.0-rc4

Published: pending

Release type:

- prerelease
- `single-node` remains the candidate GA path
- `distributed` remains beta, but its live-migration path is now hosted-validated

## Highlights

- distributed live node admission and local-to-distributed migration now pass the hosted `Release Validation` workflow
- the Storage page now reports:
  - `Pending Local Bytes`
  - `Distributed Bytes`
  - recent migration backlog by bytes as well as object count
  - a `Local drain complete` readiness signal
- HarborShield now blocks demoting the last healthy active distributed node while local drain is still incomplete
- the distributed beta runbook now documents the supported drain-complete signal and last-active-node safeguard

## Operator Notes

- start the published bundle with:

```bash
docker compose --env-file .env --env-file release-images.env up -d
```

- rotate every secret in `.env` before exposing the stack beyond local testing
- `distributed` deployment remains beta and is not part of the GA promise for this release line
- use [`../distributed-operations.md`](../distributed-operations.md) as the supported runbook for live node admission and migration
- do not demote the last healthy active node until the Storage page reports `Local drain complete`
