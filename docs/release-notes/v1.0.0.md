# HarborShield v1.0.0

Published: pending

Release type:

- stable
- `single-node` is now the signed GA path
- `distributed` remains beta

## Highlights

- HarborShield `v1.0.0` ships with a signed GA scope for `single-node`
- the `v1.0` S3 contract is now explicitly documented as a practical, non-parity compatibility promise
- published-bundle install, first-run, core S3, backup, restore, and distributed beta workflow evidence have all been captured
- GitHub CI, deep release validation, and tagged publishing are all in place for the release path

## GA Scope

`v1.0.0` GA covers:

- single-host Docker Compose deployment
- local encrypted object storage
- first-run bootstrap and password rotation
- core admin workflows
- the `Supported` S3 surface documented in [`../s3-compatibility.md`](../s3-compatibility.md)
- published release bundles, images, and checksums

Signed release-decision documents:

- [`../v1-scope.md`](../v1-scope.md)
- [`../v1-s3-contract.md`](../v1-s3-contract.md)
- [`../v1-operator-sweep.md`](../v1-operator-sweep.md)

## Beta Scope

`distributed` remains beta for this release line.

That includes:

- live node admission
- local-to-distributed migration
- distributed recovery beyond operator-managed guidance
- deeper repair and rebalance automation

Use [`../distributed-operations.md`](../distributed-operations.md) as the supported runbook for distributed beta operation.

## Validation Evidence

- published-bundle acceptance pass exists for the current release-candidate line in [`../release-acceptance/v1.0.0.md`](../release-acceptance/v1.0.0.md)
- hosted distributed validation passed in the release lane before this GA decision
- fast CI and deep release validation are green prerequisites for the tagged release workflow

## Known Boundaries

- HarborShield does not claim full Amazon S3 parity
- unsupported S3 APIs remain outside the `v1.0` contract
- `distributed` remains beta and is not part of the GA promise

## Operator Notes

- start the published bundle with:

```bash
docker compose --env-file .env --env-file release-images.env up -d
```

- rotate every secret in `.env` before exposing the stack beyond local testing
- use the single-node deployment path for the signed GA experience
- use distributed mode only with the explicit understanding that it remains beta
